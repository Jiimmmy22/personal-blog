时隔一年，agent 真的越来越成熟了。

过去，agent 给你的更像是一间毛坯房。主体结构有了，但还需要自己装修、自己验收、自己排查问题。现在的 agent 更像是直接交付一套成品房：家具、装修、动线基本一应俱全。当然，前提是你需要给出足够充分、足够清晰的 prompt。

## 一年前的 Vibe Coding

一年前的 Vibe Coding，更多还是依赖 Cursor 这类 IDE。你需要自己审代码，自己部署到 GitHub。代码一旦报错，对我这样的新手来说，最熟悉的处理方式就是截图问 AI，然后在浏览器、IDE、终端之间来回切换。

那种流程非常繁琐。

当时的 Vibe Coding 其实仍然需要一定的架构基础。你至少要能读懂报错发生在哪里，知道它大概属于依赖问题、路径问题、配置问题，还是部署问题。否则就会在一个又一个错误弹窗里绕圈。

2025 年 4 月 23 日，我心血来潮，想用 Cursor 做一个 Hexo blog。最后耗时 5 小时。我需要先去 Hexo 找 theme，把链接复制给 Cursor，然后不断重复“报错、截图、询问、修改”的循环。

也许当时是我的 prompt 不够好。总之，那一整个下午几乎都在截屏问 AI，丝毫没有乐趣。

## 现在的 Agent

如今 Codex 和 Claude Code 这类工具已经完善了很多。步骤明显更简单，也更接近“把需求交给一个能执行的协作者”。

你只需要像甲方一样提出要求，给出足够充分的 prompt。它可以自己搭架构、写组件、补 UI、修报错，甚至在内置浏览器里完成部署。

这件事的变化很大。

以前我是在“帮 AI 完成工作”。现在更像是“让 AI 帮我完成工作”。我仍然需要表达审美、方向和需求，但很多具体、繁琐、容易打断热情的步骤，已经可以交给 agent 处理。

## 这个 Blog 的诞生

2026 年 5 月 17 日凌晨，我又一次心血来潮。

我刚充了 ChatGPT Plus，想着不用白不用，又想起了一年前那个没有做顺的 blog project。于是我先让 ChatGPT 帮我生成 prompt，再把这些 prompt 分步交给 Codex。

这一次，只用了 30 分钟左右。

Codex 从网站架构开始，一步步完成页面、动效、博客系统、UI 优化和 GitHub Pages 部署。最后的部署阶段，也是它在内置浏览器中帮我完成的。

全程我做的事情，几乎只是复制 prompt、提出修改意见、确认方向。

于是，这个 blog 就这样应运而生。

## 从工具到伙伴

一年时间里，Vibe Coding 的门槛下降了很多。

过去它要求你能读懂一间毛坯房的结构图。现在，它更像是先给你一套能住的房子，再允许你慢慢调整灯光、家具和墙面颜色。

这并不意味着人不重要了。

恰恰相反，人的表达、审美、判断和目标变得更重要。因为当执行变得便宜，真正决定作品质感的，就不再只是会不会写代码，而是你是否知道自己想要什么。
